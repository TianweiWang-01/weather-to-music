# weather to music
Tianwei Wang 605175
SkyMelody is an immersive weather-based writing studio. I built it as a browser-based creative workspace where users can choose a weather mood and write in an atmosphere that matches that mood. Instead of being just a normal text editor, each weather option changes the background, mood description, music suggestions, and AI assistant prompt, so the page feels more like a small themed writing room.
The application has a writing board with local autosave, live word and character counts, focus mode, and a countdown timer. The timer supports both preset times and custom minute/second input. When the countdown ends, the page gives the user a clear reminder with particle effects, a glowing timer ring, and a flashing border around the writing area. This helps bring the user’s attention back to the draft.
I also implemented a current weather feature. When the user clicks the location button, the app uses the browser Geolocation API to get the user's latitude and longitude, then requests live weather data from Open-Meteo. The returned weather code is converted into one of the app's weather rooms, so the background, mood, AI context, and music recommendations can automatically match the user's real weather. This was integrated from the feature I built for my Website 2 assignment.
I also added weather-matched music recommendations by using the iTunes Search API. When the user changes the weather mood, the app searches for songs that fit that mood and displays the track name, artist, cover image, and playable preview if available. The preview is usually around 30 seconds because iTunes only provides short preview clips, not full songs. If the API does not respond, the app still shows local fallback recommendations, so the music section will not look empty.
For the AI writing companion, the frontend sends the current title, draft content, word count, selected weather room, and mood information to a small Node.js backend. The backend protects the OpenAI API key, forwards the request to the OpenAI API, and returns writing-focused suggestions to the page. To make the assistant better match the purpose of the website, I built a system prompt that helps the AI understand the user's current draft, title, emotional tone, sensory details, and weather-based writing room. This makes the assistant's responses more useful for brainstorming, revision, continuation, titles, and feedback instead of feeling like generic chatbot answers. This design allows the frontend to be hosted on GitHub Pages while keeping the private API key on the server side.
